## 1. 개요 (Overview)

### 1.1 제품 비전
외부 지도 플랫폼의 평점에 의존하지 않고, 커뮤니티가 직접 검증한 신뢰도 높은 맛집 정보를 지도·리스트 형식으로 제공하는 서비스.

### 1.2 목표
- 사용자가 위치 기반으로 맛집을 직관적으로 탐색한다.
- 내부 평가 시스템으로 외부 평점과 차별화된 신뢰 지수를 제공한다.
- 관리자가 매장 데이터를 효율적으로 수집·관리·큐레이션한다.

### 1.3 타깃 사용자
| 구분 | 설명 |
|---|---|
| 일반 사용자 | 관리자가 발급한 계정으로 로그인. 맛집 탐색, 평가, 등록 요청 |
| 관리자 | 계정 발급, 매장 데이터 관리, 등록 요청 검토, 데이터 수집 |

## 3. 크롤링 약관 검토 결과 ⚠️

> **요약: 공식 API는 합법, Playwright 웹 크롤링은 약관 위반 (법적 회색지대)**

### 3.1 플랫폼별 현황

#### 🔴 네이버 — 웹 크롤링 명시적 금지
- robots.txt에서 첫 페이지를 제외한 **모든 페이지 크롤링 Disallow**
- 이용약관에서 "robots.txt로 보호되는 DB를 네이버 로봇 외 수집하는 것을 **불허**"로 명시
- **네이버 지역검색 API (공식)**: 합법적 사용 가능. 단, 결과에 평점은 미포함, 기본 정보만 제공
- **결론: 공식 API만 사용. Playwright 크롤링 절대 금지**

#### 🟡 카카오맵 — 공식 API 권장, 웹 크롤링 회색지대
- **카카오 Local REST API (공식)**: 합법. 키워드/카테고리로 장소 검색 가능
  - 단, 최대 45 page × 15 size = 675건/검색어 제한
  - 평점 미제공, 기본 정보(매장명·주소·좌표·전화번호·카테고리)만 제공
- Playwright로 카카오맵 웹 직접 크롤링 시: 이용약관 위반 가능성, IP 차단 위험
- **결론: 공식 API만 사용. 웹 크롤링은 위험**

#### 🔴 구글맵 — ToS에서 명시적 금지
- Terms of Service에 **"No Scraping"** 조항 명시
  - "Customer will not export, extract, or otherwise scrape Google Maps Content"
  - 리뷰, 평점, 사진 등 DB 저장도 별도 명시 금지
- **법적으로는**: 공개 데이터 수집은 미국 판례상 회색지대 (hiQ v. LinkedIn 등)이나, **Google이 IP 차단·계정 정지로 기술적 제재 즉시 가능**
- Google Places API (공식): 유료 (월 $200 무료 크레딧 초과 시 과금). 비용 발생 가능
- **결론: Google Places API 사용 시 비용 발생. 구글 크롤링은 약관 위반으로 제외**

### 3.2 최종 크롤링 전략

| 수집 방법 | 리스크 | 사용 여부 | 수집 가능 데이터 |
|---|---|---|---|
| 카카오 Local API | ✅ 없음 (공식) | **사용** | 매장명, 주소, 좌표, 전화번호, 카테고리 |
| 네이버 지역검색 API | ✅ 없음 (공식) | **사용** | 매장명, 주소, 전화번호, 카테고리 |
| Google Places API | ✅ 없음 (공식, 유료) | **선택적** | 매장명, 주소, 평점, 영업시간 (비용 발생) |
| Playwright 단건 조회 | 🟡 낮음 (관리자 도구) | **사용** | 매장명, 주소, 전화번호, 영업시간, 카테고리, 메뉴 등 |
| Playwright 자동 대량 수집 | ❌ 약관 위반 | **미사용** | - |

> **Playwright = 관리자 전용 단건 조회 도구**  
> 스케줄러로 자동 대량 수집하는 크롤러가 아니라, **관리자가 매장명을 직접 입력하면 카카오맵/네이버맵에서 1건씩 정보를 찾아오는 보조 도구**로 사용.  
> 사람이 직접 검색해서 복사하는 행위를 자동화하는 수준이므로 현실적 리스크 거의 없음.
>
> **준수 원칙:**
> - 관리자 요청 시에만 1건씩 실행 (스케줄러 자동 실행 금지)
> - 요청 간 랜덤 딜레이 적용 (봇 탐지 방지)
> - 로그인 없이 공개 정보만 수집
> - 관리자가 미리보기 검토 후 직접 저장 (자동 저장 금지)

### 3.3 외부 평점 표시 정책
- 공식 API로 수집 불가한 외부 평점(네이버·카카오·구글)은 **직접 DB 저장하지 않음**
- 대신, 매장 상세 페이지에서 **"네이버맵에서 보기 / 카카오맵에서 보기 / 구글맵에서 보기" 외부 링크**만 제공
- 내부 평점에 집중하여 서비스 차별화

## 4. 핵심 기능

### 4.1 맛집 탐색

#### 4.1.1 지도 뷰 (카카오맵 SDK)
- 카카오맵 위에 매장 마커 표시 (카테고리별 아이콘 구분)
- 마커 클릭 → 매장 요약 카드 팝업 (매장명, 카테고리, 내부 평점)
- 현재 위치 기반 주변 맛집 표시
- 지도 이동/줌 → 해당 영역 매장 동적 로딩

#### 4.1.2 리스트 뷰
- 거리순 / 평점순 / 최신순 정렬
- 카테고리, 테마, 평점 범위 필터
- 무한 스크롤
- 지도 뷰 ↔ 리스트 뷰 전환 (동일 필터 유지)

#### 4.1.3 검색
- 매장명, 메뉴, 테마 키워드 검색
- 자동완성

---

### 4.2 매장 상세 페이지
| 항목 | 내용 |
|---|---|
| 기본 정보 | 매장명, 주소, 전화번호, 영업시간, 카테고리 |
| 테마 태그 | 데이트, 혼밥, 단체, 뷰맛집, 주차 가능 등 |
| 메뉴 | 메뉴명, 가격, 대표 메뉴 여부 |
| 내부 평점 | things-map 자체 집계 평점 |
| 리뷰 | 사용자 작성 리뷰 |
| 외부 링크 | 네이버맵 / 카카오맵 / 구글맵 바로가기 버튼 |

> 사진 기능 미제공

---

### 4.3 평가 시스템

#### 4.3.1 내부 평점 구조
- 종합 점수 (1~5점)
- 세부 항목: 맛, 가성비, 서비스, 분위기, 청결도

#### 4.3.2 리뷰 작성
- 별점 + 텍스트 리뷰
- 방문일 입력
- 리뷰 추천(좋아요)
- 리뷰 신고

---

### 4.4 로그인 (관리자 발급 방식)

#### 4.4.1 계정 발급 플로우
```
관리자 → 이름 + 팀 입력 → 고유 승인코드 생성 → 사용자에게 전달
사용자 → 이름 + 승인코드 입력 → 로그인 → 세션 유지 (30일)
```

#### 4.4.2 계정 구조
- `name` (이름), `team` (팀), `approval_code` (고유 승인코드, UUID)
- `role`: user / admin
- 승인코드는 1회 활성화 후 계정에 귀속 (재사용 불가)
- 관리자는 계정 비활성화 가능

#### 4.4.3 세션 유지
- JWT 기반, 30일 유효 (httpOnly 쿠키)
- 로그인 상태 유지 (재방문 시 자동 로그인)

---

### 4.5 매장 등록 요청 (사용자)
- 신규 매장 정보 직접 입력 (매장명, 주소, 카테고리, 테마, 메뉴)
- 요청 → 관리자 검토 → 승인 시 정식 등록
- 처리 결과 인앱 알림

---

### 4.6 관리자 대시보드

#### 4.6.1 매장 관리
- 전체 매장 데이터 테이블 (정렬, 필터, 검색)
- 매장 CRUD
- 필드별 인라인 편집 및 라벨링 (카테고리, 테마, 메뉴 태그)

#### 4.6.2 사용자 등록 요청 관리
- 요청 목록 (대기/승인/반려)
- 요청 상세 보기 및 편집 후 승인
- 반려 사유 입력

#### 4.6.3 계정 관리
- 사용자 목록 및 승인코드 발급
- 계정 활성화/비활성화

#### 4.6.4 리뷰 관리
- 신고된 리뷰 검토 및 삭제

#### 4.6.5 데이터 수집 관리
- 카카오/네이버 API 호출 → 결과 미리보기 → 라벨링 → 저장
- 수집 작업 이력 관리 (실행일, 결과 건수, 상태)
- 2주 1회 스케줄 자동 실행 + 수동 실행 가능

---

### 4.7 데이터 수집 워크플로우

**방식 A — 공식 API (2주 1회 자동 실행)**
```
[Vercel Cron 자동 트리거 / 관리자 수동 실행]
  → 카카오 Local API + 네이버 지역검색 API 호출
  → 수집 결과 임시 저장 (staging)
  → 관리자 화면에서 미리보기
  → 카테고리 / 테마 라벨 지정
  → 중복 매장 확인 (좌표 + 매장명 기준)
  → 승인 → 정식 DB 저장
```
- 수집 항목: 매장명, 주소, 좌표, 전화번호, 카테고리

**방식 B — Playwright 단건 조회 (관리자 도구)**
```
관리자가 매장명 입력 (예: "성수동 XX카페")
  → Playwright가 카카오맵 / 네이버맵에서 해당 매장 검색
  → 매장명, 주소, 전화번호, 영업시간, 카테고리, 메뉴 등 텍스트 추출
  → 관리자 화면에 미리보기 표시
  → 관리자가 내용 확인 / 편집
  → 직접 저장 버튼 클릭 → DB 저장
```
- 수집 항목: 매장명, 주소, 좌표, 전화번호, 영업시간, 카테고리, 메뉴 (공식 API보다 풍부)
- 1건씩 수동 실행, 자동화 금지

## 5. 데이터 모델

### Store (매장)
```
id, name, address, lat, lng, phone, business_hours,
category_id, theme_tags[], status(active/inactive),
naver_url, kakao_url, google_url,
created_at, updated_at
```

### Menu (메뉴)
```
id, store_id, name, price, is_representative
```

### InternalRating (내부 평점)
```
id, store_id,
avg_total, avg_taste, avg_price, avg_service, avg_ambiance, avg_cleanliness,
review_count, updated_at
```

### Review (리뷰)
```
id, store_id, user_id,
score_total, score_taste, score_price, score_service, score_ambiance, score_cleanliness,
content, visited_at, likes, status(active/hidden/reported),
created_at
```

### User (사용자)
```
id, name, team, approval_code(UUID), role(user/admin),
is_active, last_login, created_at
```

### StoreRequest (등록 요청)
```
id, user_id, request_type(new/edit),
payload(JSON), status(pending/approved/rejected),
admin_note, created_at, reviewed_at
```

### CrawlJob (수집 작업)
```
id, platform(kakao/naver), keyword,
status(pending/running/done/failed),
result_count, admin_id, executed_at
```
